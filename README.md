# Обнаружение лесных пожаров по спутниковым снимкам с помощью CNN

## 1. Проблема и контекст

Лесные пожары наносят серьёзный ущерб экосистемам, инфраструктуре и здоровью людей. Важнейшая задача служб мониторинга — как можно раньше обнаруживать очаги возгорания на больших территориях. Традиционный визуальный контроль (наблюдательные башни, обходы) не масштабируется на большие площади и сильно зависит от человеческого фактора.

Спутниковые снимки позволяют регулярно получать изображения больших территорий, но их ручной просмотр специалистами занимает много времени. Поэтому актуальна задача автоматического анализа спутниковых снимков с помощью моделей машинного обучения, чтобы быстро выделять зоны с признаками пожара и передавать их на проверку операторам. [web:115][web:120]

## 2. Решение (ИТ‑система)

В проекте реализован прототип модуля автоматического обнаружения лесных пожаров по спутниковым изображениям на основе сверточной нейронной сети (CNN). Модуль выполняет следующие функции:

- принимает на вход снимки участков местности;
- классифицирует каждый снимок как **“пожар”** или **“нет пожара”**;
- может быть встроен в систему мониторинга для приоритезации участков, требующих проверки.

Архитектура CNN основана на публичном решении с Kaggle (“Wild Fire CNN Accuracy 95%”), адаптирована и упрощена для учебного проекта: сохранена общая идея нескольких блоков свёрток и полносвязных слоёв, но модель сделана легче и обучается на уменьшенных по размеру изображениях. [file:145][web:120]

## 3. Датасет

Используется датасет **Wildfire Prediction Dataset (Satellite Images)** c Kaggle. [web:115]

- Источник: Kaggle — `Wildfire Prediction Dataset (Satellite Images)`.  
- Формат данных: спутниковые изображения в цвете (RGB), размер исходных файлов около 350×350 пикселей.  
- Структура папок:
  - `train/` — обучающая выборка,
  - `valid/` — валидационная выборка,
  - `test/` — тестовая выборка,
  - внутри каждой папки — две классовые директории: `wildfire` и `nowildfire`. [file:145][web:115]

Для обучения в ноутбуке изображения масштабируются до фиксированного размера (например, 64×64 или 32×32 пикселя) и нормализуются (деление значений пикселей на 255), чтобы сеть работала быстрее и устойчивее. [web:146][web:151]

Целевая переменная — бинарный класс:

- `1` — на снимке есть признаки пожара (`wildfire`);
- `0` — пожара нет (`nowildfire`). [file:145]

## 4. Модель и методы

### Архитектура CNN

Модель реализована с помощью `TensorFlow / Keras` как последовательная CNN:

- несколько сверточных слоёв `Conv2D` с активацией `ReLU` и ядрами 3×3 для выделения локальных признаков (огонь, дым, текстура леса);  
- слои `BatchNormalization` для стабилизации обучения;  
- слои `MaxPooling2D` для уменьшения пространственного разрешения и выделения более абстрактных признаков;  
- слой `Flatten` и несколько полносвязных слоёв `Dense` c `ReLU`;  
- слой `Dropout` для снижения переобучения;  
- выходной слой `Dense(1, activation="sigmoid")` для предсказания вероятности класса “пожар”. [file:145]

Обучение:

- функция потерь: `binary_crossentropy`;  
- оптимизатор: `Adam`;  
- метрика: `accuracy` (дополнительно в ноутбуке можно считать precision/recall/F1). [web:150][web:151]  
- небольшое число эпох (например, 5) — достаточно для демонстрации обучения и получения разумного качества в рамках учебного проекта.

### Предобработка и аугментация

- Нормализация: `rescale=1./255` для перевода пикселей из диапазона 0–255 к 0–1. [web:146][web:151]  
- Для обучающей выборки используется `ImageDataGenerator` с простыми аугментациями (повороты, сдвиги, масштабирование, отражения), что помогает модели лучше обобщать на разные условия освещения и ракурсы. [web:149][web:152]

## 6. Установка и запуск

### Зависимости

Минимальный набор:

pip install tensorflow numpy matplotlib


Если данные читаются напрямую из локальных папок с помощью `ImageDataGenerator`, достаточно стандартного `tensorflow` и `Pillow` (обычно входит в зависимости TensorFlow). [web:149][web:153]

### Шаги запуска

1. Скачайте датасет **Wildfire Prediction Dataset (Satellite Images)** с Kaggle и распакуйте папки `train`, `valid`, `test` (или укажите путь к ним в ноутбуке). [web:115]  
2. Откройте Jupyter Notebook `wildfire_cnn_project.ipynb`.  
3. Убедитесь, что пути к данным (`train_dir`, `val_dir`, `test_dir`) указаны верно. [file:145]  
4. Запустите ячейки по порядку:
   - импорт библиотек,
   - подготовка генераторов данных,
   - определение и компиляция модели,
   - обучение,
   - оценка и визуализация результатов.  

## 7. Результаты и выводы

В ноутбуке фиксируются:

- точность модели на валидационной и тестовой выборках;  
- графики изменения `loss` и `accuracy` по эпохам;  
- несколько примеров спутниковых снимков с истинными и предсказанными метками классов.

На основе экспериментов делается вывод о том, что:

- CNN‑модель способна с высокой точностью отличать снимки с пожарами от снимков без пожаров на выбранном датасете; [web:120]  
- такой модуль можно использовать как компонент системы мониторинга, которая автоматически отмечает потенциально опасные участки для дальнейшей проверки специалистами;  
- качество и устойчивость решения можно дополнительно улучшить за счёт:
  - использования более сложной архитектуры или предобученной сети (transfer learning),
  - увеличения числа обучающих данных,
  - более продвинутых аугментаций и подбора гиперпараметров. [web:150][web:151]
